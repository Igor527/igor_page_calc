# PWA и индексация главной страницы

## Почему Google мог считать главную страницу редиректом

1. **Канонический URL**  
   В `index.html` добавлен `<link rel="canonical" href="https://urbanplanner.page/">`, чтобы поисковик считал один основной адрес главной и не путал версии (http/https, с www и без).

2. **Редиректы на хосте**  
   Проверь в панели хостинга (Cloudflare Pages / Vercel / и т.д.):
   - нет ли правил редиректа с `/` на другой URL (например `/?utm_` или `/home`);
   - редирект с `http` на `https` и с `www` на без `www` (или наоборот) — это нормально, но тогда canonical должен указывать на итоговый URL, с которого отдаётся страница.

3. **SPA и `_redirects`**  
   В проекте используется правило `/* /index.html 200` (rewrite, не 302). Для робота запрос к `/` должен возвращать 200 и тот же URL. Если у тебя другое правило (например 302 на `/index.html`), замени на 200 rewrite.

4. **Проверка в Search Console**  
   В Google Search Console → Проверка URL → вставь `https://urbanplanner.page/` и посмотри, какой ответ и редирект видит Google. При необходимости запроси переобход.

## Почему «Установить» в Chrome иногда появляется, иногда нет

Критерии показа кнопки «Установить приложение» в Chrome:

- Валидный `manifest.json` с `name`, `start_url`, `icons`.
- Иконки: для стабильного появления кнопки нужны PNG 192×192 и 512×512. Они **генерируются автоматически** из `public/icon.svg` (буквы UP в стиле Pico) при сборке: скрипт `scripts/generate-pwa-icons.mjs` запускается по `prebuild` и создаёт `public/icon-192.png` и `public/icon-512.png`. Менять дизайн нужно только в `icon.svg`.
- Сервис-воркер не обязателен для показа установки, но его наличие может влиять на кеш и поведение.

## Название иконки приложения (раньше «IP»)

- В **`public/icon.svg`** в качестве текста на иконке было «IP» (Igor.Page). Заменено на **«UP»** (Urban Planner).
- В **`index.html`** мета-тег `apple-mobile-web-app-title` был `Igor.Page`; заменён на **Urban Planner**, чтобы название при добавлении на экран домашний (iOS/Android) совпадало с брендом.

После деплоя обнови страницу с очисткой кеша или переустанови приложение, чтобы подтянулись новый манифест и иконка.
