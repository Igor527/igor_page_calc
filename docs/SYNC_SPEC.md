# ТЗ: синхронизация между устройствами и ОС

## Цель

Обеспечить согласованность данных (блог, заметки, словарь, калькуляторы, порядок секций, планировщик) при работе админа с разных устройств (ПК, Android, iOS) и ОС без потери правок и удалений.

## Источник истины

- **Репо GitHub** (`public/data/*.json`) — каноническое хранилище для деплоя и синхронизации.
- **localStorage** на каждом устройстве — локальный кэш; при наличии настроенной синхронизации изменения пушатся в репо (pull → merge → push для блога; прямой push для остальных).

## Сценарии

| Сценарий | Ожидание |
|----------|----------|
| Правка с телефона, затем с ПК | Оба набора правок сохраняются; при конфликте по одному объекту выигрывает версия с более поздним `updatedAt` (или аналог). |
| Удаление с телефона, пуш с ПК | Удаление не теряется: используется мягкое удаление (флаг `deleted`) и слияние по времени. |
| Нет сети на одном устройстве | Локальные изменения остаются в localStorage; при появлении сети — ручная или автоматическая синхронизация. |
| Разные ОС (Windows, macOS, Android, iOS) | Один и тот же фронт и API GitHub; различия только в браузере (localStorage, cookies). На мобильных — разрешить данные сайта, не использовать режим инкогнито для сохранения настроек синхронизации. |

## Риск затирания (кто последний пушит — тот перезаписывает файл)

| Сущность        | Перед push загружаем репо? | Merge по версиям? | Затирание при двух устройствах |
|-----------------|----------------------------|-------------------|---------------------------------|
| **Блог (posts)**| Да                         | Да (по `updatedAt` + `deleted`) | Нет: всегда pull → merge → push |
| Заметки         | Нет                        | Нет               | Да: последний push перезаписывает файл |
| Словарь         | Нет                        | Нет               | Да |
| Калькуляторы    | Нет                        | Нет               | Да |
| Порядок окон    | Нет                        | Нет               | Да |
| Планировщик     | Нет                        | Нет               | Да |

**Рекомендация:** с двух устройств одновременно по возможности редактировать только блог (он защищён). Заметки, словарь, layouts, planner, калькуляторы — лучше менять с одного устройства за раз либо перед правкой на втором устройстве один раз нажать «Синхронизировать с репо» в блоге (для остальных сущностей отдельной кнопки «подтянуть с репо» пока нет).

## Реализованные механизмы

### Блог (posts.json)

- **Pull перед push**: перед отправкой загружаем текущий `posts.json` из репо.
- **Слияние**: по каждому посту (по `id`) берётся версия с большим `updatedAt`; удаления сохраняются через поле `deleted`.
- **Пауза 6 сек** перед автоматической отправкой; кнопки «Отправить сейчас», «Отменить», «Синхронизировать с репо» (только загрузка и слияние без push).

### Остальные сущности (notes, dictionary, calculators, layouts, planner)

- Прямой push при изменениях (с дебаунсом). Конфликты не мержатся автоматически; при необходимости — вручную подтянуть актуальный файл из репо или настроить merge позже.

### Настройки синхронизации (owner, repo, token)

- Хранятся в **localStorage** (токен — только там, не в env).
- Owner/repo/branch можно задать через env (`VITE_GITHUB_SYNC_*`) для подстановки в форму; на новом устройстве токен вводится один раз.

## Ограничения и рекомендации

- **Один админ** или согласованная работа: при одновременном редактировании одного и того же поста с двух устройств побеждает последнее сохранённое по времени.
- **Мобильные**: включить «Файлы cookie и данные сайта» для сайта; для надёжного сохранения настроек не использовать режим инкогнито.
- **Резервные копии**: при необходимости делать экспорт (например, «Экспорт для GitHub» для блога) или бэкапить репо.

## Дальнейшее развитие (опционально)

- Merge по конфликтам для notes/dictionary (по id с `updatedAt`).
- Индикатор «есть несохранённые изменения» и явная кнопка «Синхронизировать всё».
- Офлайн-очередь с повторными попытками push при восстановлении сети.
