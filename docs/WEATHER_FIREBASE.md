# Метеостанция: лог в Firebase (без CORS)

Данные из Google Таблицы отправляются в Firebase Realtime Database скриптом. Сайт читает лог по REST — без CORS и без публичной ссылки на таблицу.

## Шаги

### 1. Firebase

1. [Firebase Console](https://console.firebase.google.com/) → создать проект (или использовать существующий).
2. В проекте: **Realtime Database** → **Создать базу** (режим можно тестовый, потом подправите правила).
3. Скопировать **URL базы** (вид: `https://ПРОЕКТ-default-rtdb.firebaseio.com`).

### 2. Секретный ключ

Сгенерировать длинный ключ (тот же будет в таблице и в .env сайта), например:

```bash
openssl rand -hex 32
```


Или любой пароль 20+ символов.

### 3. Правила Realtime Database

В Firebase → Realtime Database → **Правила** задать (подставьте свой путь с вашим ключом — или откройте доступ ко всему `weather` для простоты):

```json
{
  "rules": {
    "weather": {
      "$dataKey": {
        ".read": true,
        ".write": true
      }
    }
  }
}
```

Путь к данным: `weather/ВАШ_КЛЮЧ/rows/`. Только тот, кто знает `ВАШ_КЛЮЧ`, сможет читать/писать. Не используйте один и тот же ключ в публичных репозиториях.

### 4. Сайт (.env)

В `.env` (или в секретах сборки на Cloudflare/Vercel):

```env
VITE_FIREBASE_DATABASE_URL=https://ПРОЕКТ-default-rtdb.firebaseio.com
VITE_WEATHER_DATA_KEY=ваш_секретный_ключ
```

Опционально: `VITE_WEATHER_STATION_ID=название` для подписи.

### 5. Google Таблица и скрипт

1. Откройте вашу таблицу с логом (дата, температура, давление, PM и т.д.).
2. **Расширения** → **Apps Script**.
3. Вставьте код из репозитория: `scripts/weather-sheet-to-firebase.gs`.
4. **Файл** → **Настройки проекта** → вкладка **Скрипты**:
   - `FIREBASE_DB_URL` = URL базы (как в .env).
   - `WEATHER_DATA_KEY` = тот же ключ, что в .env.
5. Сохраните и первый раз запустите вручную: выберите функцию **syncWeatherToFirebaseFull** → **Выполнить** (отправит все строки).
6. **Триггер**: нажмите **Триггеры** (часовик слева) → **Добавить триггер** → функция `syncWeatherToFirebase`, событие **По времени** (например, каждые 5 или 15 минут).

Дальше скрипт будет отправлять только **новые** строки при каждом запуске.

### 6. Усреднение и годы данных

- Сейчас в приложении берутся последние **10 000** точек по дате. Этого хватает на месяцы/год при записи раз в 5–15 минут.
- Для хранения лет лога можно позже добавить в скрипт или в Cloud Function расчёт средних по часу/дню и запись в `weather/KEY/aggregated/daily/YYYY/MM`, а на сайте — выбор «по дням» и чтение только агрегатов.

## Формат строки в Firebase

Каждая запись в `weather/KEY/rows/{pushId}`:

- `date` — число (timestamp, мс)
- `temperature`, `pressure`, `pm1`, `pm25`, `pm10` — числа или не заданы
- `station` — строка (опционально)

Скрипт сам определяет колонки по заголовкам (date, temperature, давление, pm2.5, pm10 и т.п.) или по фиксированным позициям для формата AirStationLog.
