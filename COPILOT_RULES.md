# COPILOT_RULES.md — Универсальные правила для проектов Igor.Page Engine

## Структура и документация
- Всегда поддерживайте .gitignore, .env.example, README.md, TODO.md, CHANGELOG.md, PROJECT_GUIDELINES.md, COPILOT_RULES.md.
- Документируйте структуру, бизнес-логику и workflow в отдельных markdown-файлах.

## Соглашения об именовании
- camelCase — переменные, функции, id элементов.
- PascalCase — классы.
- UPPER_SNAKE_CASE — константы.
- kebab-case — CSS классы.
- data-kebab-case — data-атрибуты.

## Архитектурные принципы
- Нет хардкодов: все тексты, метки, параметры — из единого источника данных (schema, blocks, data).
- Все вычисления и формулы — только через движок (engine.ts), без дублирования логики.
- Все селекторы, списки, опции — из данных, а не вручную.

## Код-стайл и логирование
- Проверять наличие элементов DOM перед использованием.
- Использовать console.log/warn/error с префиксами (✅, ❌, ⚠, 📊, 🔍, 🔧).
- Все асинхронные операции — через async/await, с обработкой ошибок.
- Math.ceil для округлений (если применимо).
- JSDoc-комментарии для функций.
- Логичные разделы кода — через комментарии.

## UI/UX
- Все визуальные элементы (кнопки, селекторы) — с placeholder-опциями, единым стилем.
- Визуальная обратная связь (цвета, подсветка) для действий пользователя.
- Строгий порядок загрузки скриптов и данных.
- Использовать PicoCSS, тёмная тема по умолчанию, header/footer и кнопки — как в PROJECT_GUIDELINES.md.

## Тестирование
- Проверять консоль на ошибки.
- Проверять заполнение селекторов из данных.
- Проверять корректность расчётов и отображения.

## Специальные правила
- Не повторять отладочные сообщения.
- Все ответы и комментарии — на русском языке.
- Всегда проверять и дополнять этот файл при изменениях.

## Дополнительно
- Валидация входных данных.
- Сохранение/восстановление данных (localStorage).
- Экспорт результатов (PDF/Excel) — по мере необходимости.
