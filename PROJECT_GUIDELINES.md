# Общие инструкции для всех проектов

1. Всегда добавляйте .gitignore и .env.example в корень проекта.
2. Все приватные ключи, токены, пароли и секреты храните только в .env (не коммитьте в репозиторий).
3. Документируйте структуру проекта и бизнес-логику в отдельных markdown-файлах (.project-structure.md, .project-instructions.md, WORKFLOW.md).
4. Используйте README.md для описания назначения, установки и запуска проекта.
5. Ведите TODO.md и CHANGELOG.md для планирования и отслеживания изменений.
6. Для UI используйте единый стиль и компоненты (см. .project-instructions.md).
7. Для деплоя используйте Cloudflare Pages, Vercel или аналогичные сервисы.
8. Для кастомных доменов настройте DNS и укажите домен в настройках деплоя.
9. Всегда обновляйте .env.example при добавлении новых переменных.
10. Проверяйте, что все зависимости и инструкции по запуску актуальны.

## 🔒 БЕЗОПАСНОСТЬ (ОБЯЗАТЕЛЬНО ДЛЯ ВСЕХ ПРОЕКТОВ)

### Критические требования безопасности

#### 1. Защита от XSS
- **НИКОГДА не используйте `innerHTML` или `dangerouslySetInnerHTML` без санитизации**
- Все пользовательские данные экранируются перед отображением
- Используйте функции санитизации из `src/lib/security.ts`

#### 2. Валидация URL
- **ВСЕ URL проверяются перед использованием**
- Разрешайте только безопасные протоколы: `http:`, `https:`, `data:` (только для изображений)
- Блокируйте `javascript:`, `vbscript:`, `data:text/html`

#### 3. Защита от скрытых вызовов к БД/API
- **КРИТИЧНО:** При обработке текстовых данных (чтение/запись, парсинг JSON, обработка формул) **НИКОГДА не выполняйте скрытые вызовы к базам данных или внешним API**
- Все операции с данными должны быть явными и прозрачными
- Не используйте `eval()`, `Function()`, `import()`, `require()` в пользовательских данных
- Все формулы валидируются перед выполнением

#### 4. Валидация всех пользовательских данных
- **ВСЕ данные от пользователя валидируются перед использованием**
- Используйте функции валидации из `src/lib/validation.ts`
- Проверяйте типы, обязательные поля, зависимости

#### 5. Защита формул
- Все формулы проверяются на опасные паттерны
- Используйте только безопасные библиотеки (например, `math.js`)
- Никогда не передавайте пользовательские данные напрямую в `evaluate()` без валидации

#### 6. Защита при сохранении/загрузке
- Все данные валидируются перед сохранением
- Все данные валидируются при загрузке
- При ошибке валидации — не загружать данные

### Обязательные файлы безопасности

При создании нового проекта обязательно добавьте:

1. **src/lib/security.ts** — утилиты безопасности:
   - Санитизация HTML
   - Валидация URL
   - Валидация формул
   - Проверка опасного кода

2. **src/lib/validation.ts** — валидация данных:
   - Валидация схемы
   - Проверка зависимостей
   - Обнаружение циклических зависимостей

3. **SECURITY.md** — документация по безопасности:
   - Описание реализованных мер
   - Best practices
   - Известные ограничения

### Что реализовано в этом проекте (igor_page_calc)

✅ **src/lib/security.ts** — полный набор утилит безопасности:
- `sanitizeHtml()`, `escapeHtml()`, `sanitizeText()`
- `isValidUrl()`, `sanitizeUrl()`
- `isValidFormula()`, `containsDangerousCode()`
- `isValidBlockId()`

✅ **src/lib/validation.ts** — валидация схемы:
- `validateBlocks()` — полная валидация
- `validateImportedBlocks()` — валидация при импорте
- Проверка уникальности, зависимостей, циклов

✅ **Защита в компонентах:**
- Санитизация текста в `ReportPanel.tsx`
- Валидация URL изображений
- Валидация при импорте JSON в `BlueprintPanel.tsx`
- Исправлен `innerHTML` в `page.html`

✅ **Защита в store:**
- Валидация при сохранении/загрузке из localStorage

**Подробности:** См. `SECURITY.md` и раздел безопасности в `COPILOT_RULES.md`
