# Общие инструкции для всех проектов

1. Всегда добавляйте .gitignore и .env.example в корень проекта.
2. Все приватные ключи, токены, пароли и секреты храните только в .env (не коммитьте в репозиторий).
3. Документируйте структуру проекта и бизнес-логику в отдельных markdown-файлах (.project-structure.md, .project-instructions.md, WORKFLOW.md).
4. Используйте README.md для описания назначения, установки и запуска проекта.
5. Ведите TODO.md и CHANGELOG.md для планирования и отслеживания изменений.
6. Для UI используйте единый стиль и компоненты (см. .project-instructions.md).
7. Для деплоя используйте Cloudflare Pages, Vercel или аналогичные сервисы.
8. Для кастомных доменов настройте DNS и укажите домен в настройках деплоя.
9. Всегда обновляйте .env.example при добавлении новых переменных.
10. Проверяйте, что все зависимости и инструкции по запуску актуальны.

Перед стартом нового проекта:
- Скопируйте этот файл и следуйте этим правилам.
- Проверьте наличие всех обязательных файлов и инструкций.
