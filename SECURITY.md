# Безопасность проекта Igor Page Calc

## Реализованные меры защиты

### 1. Защита от XSS (Cross-Site Scripting)

#### Санитизация HTML
- **Файл:** `src/lib/security.ts`
- **Функции:** `sanitizeHtml()`, `escapeHtml()`, `sanitizeText()`
- **Использование:**
  - Все текстовые блоки санитизируются перед отображением
  - HTML-теги экранируются, разрешены только безопасные теги (b, i, u, strong, em, br, p, span)
  - Переносы строк преобразуются в `<br>` безопасным способом

#### Исправление innerHTML
- **Файл:** `src/app/page.html`
- **Было:** Использование `innerHTML` для вставки header/footer
- **Исправлено:** Использование `createElement` и `textContent` для безопасного создания DOM-элементов
- **Дополнительно:** Добавлен `rel="noopener noreferrer"` для внешних ссылок

### 2. Валидация URL

#### Проверка протоколов
- **Файл:** `src/lib/security.ts`
- **Функции:** `isValidUrl()`, `sanitizeUrl()`
- **Разрешённые протоколы:**
  - `http:`, `https:` — для обычных ссылок
  - `data:` — только для изображений (png, jpeg, gif, svg, webp)
- **Блокируются:**
  - `javascript:`, `vbscript:` — выполнение кода
  - `data:text/html` — HTML в data URI
  - Другие опасные протоколы

#### Валидация в блоках image
- **Файл:** `src/components/editor/ReportPanel.tsx`
- Все URL изображений проверяются перед отображением
- Небезопасные URL блокируются с показом сообщения об ошибке

### 3. Валидация формул

#### Защита от вредоносного кода
- **Файл:** `src/lib/security.ts`
- **Функция:** `isValidFormula()`
- **Блокируется:**
  - `import()`, `require()` — импорт модулей
  - `eval()` — выполнение произвольного кода
  - `Function()` конструктор
  - Доступ к `constructor`, `__proto__`, `prototype`

#### Проверка при сохранении
- **Файл:** `src/lib/validation.ts`
- Все формулы валидируются перед сохранением в store
- Ошибки валидации логируются в консоль

### 4. Валидация схемы блоков

#### Проверка структуры
- **Файл:** `src/lib/validation.ts`
- **Функция:** `validateBlocks()`
- **Проверяется:**
  - Уникальность ID блоков
  - Валидность ID (только буквы, цифры, подчёркивания, дефисы)
  - Обязательные поля для каждого типа блока
  - Существование зависимостей (ссылки на другие блоки)
  - Циклические зависимости в формулах

#### Визуальное отображение ошибок
- **Файл:** `src/components/editor/ValidationErrors.tsx`
- Компонент показывает все ошибки валидации в BlueprintPanel
- Ошибки отображаются с указанием блока и поля

### 5. Защита при импорте JSON

#### Валидация импортируемых данных
- **Файл:** `src/lib/validation.ts`
- **Функция:** `validateImportedBlocks()`
- **Проверяется:**
  - Корректность JSON структуры
  - Наличие обязательных полей
  - Валидность всех блоков
- **Использование:** `src/components/editor/BlueprintPanel.tsx`
  - При вставке JSON через textarea
  - При импорте из файла

### 6. Защита при сохранении/загрузке

#### Валидация при сохранении
- **Файл:** `src/lib/store.ts`
- Все блоки валидируются перед сохранением в localStorage
- Ошибки логируются, но не блокируют сохранение (можно улучшить)

#### Валидация при загрузке
- **Файл:** `src/lib/store.ts`
- Данные из localStorage валидируются при загрузке
- Если валидация не прошла, загружаются пустые блоки

### 7. Дополнительные меры

#### Проверка опасного контента
- **Файл:** `src/lib/security.ts`
- **Функция:** `containsDangerousCode()`
- Проверяет наличие опасных паттернов в тексте:
  - `javascript:`, `vbscript:`
  - Обработчики событий (`onclick=`, `onerror=` и т.д.)
  - Теги `<script>`
  - Вызовы `eval()`, `import()`, `require()`

## Рекомендации для дальнейшего улучшения

### 1. Content Security Policy (CSP)
Добавить заголовки CSP в HTML для дополнительной защиты:
```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net;">
```

### 2. Санитизация на сервере
Если будет добавлен backend, валидацию нужно дублировать на сервере.

### 3. Rate Limiting
Для API запросов (если будет добавлен) нужно ограничить частоту запросов.

### 4. Логирование безопасности
Логировать все попытки импорта невалидных данных для аудита.

### 5. Тестирование безопасности
Добавить unit-тесты для всех функций безопасности.

## Известные ограничения

1. **Валидация формул:** Базовая проверка паттернов, не проверяет все возможные векторы атак
2. **Санитизация HTML:** Разрешены некоторые теги, но атрибуты удаляются (можно улучшить)
3. **Локальное хранение:** Данные хранятся в localStorage без шифрования (для публичных калькуляторов это нормально)
