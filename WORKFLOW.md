# Workflow проекта

## Основные процессы

### 1. Создание калькулятора (гость)
- Гость открывает конструктор, собирает калькулятор из блоков.
- Сохраняет схему локально (cookie/localStorage).
- Может продолжить работу, пока cookie не удалены.

### 2. Подача на публикацию
- Гость/автор отправляет калькулятор на ревью через UI.
- Калькулятор получает статус "review".

### 3. Ревью админом
- Админ видит список калькуляторов на ревью.
- Проверяет, комментирует, может отклонить или опубликовать.
- В случае ошибок — возвращает с комментариями.

### 4. Публикация
- После одобрения калькулятор становится публичным.
- Доступен по уникальной ссылке.

### 5. Редактирование и повторное ревью
- Автор может исправить ошибки и повторно отправить на ревью.

## Технический workflow
- Все изменения структуры отражать в .project-structure.md.
- Все новые бизнес-правила и паттерны — в .project-instructions.md.
- Вести README.md, CHANGELOG.md, TODO.md.
- Для деплоя использовать Cloudflare Pages.
- Для серверныхless-функций (API) — /functions или /api (если потребуется).

## AI-ассистент (Mistral AI)

- В админском интерфейсе редактора появляется третье окно (AI-панель).
- Из любого места UI можно вызвать контекстное меню для отправки запроса в Mistral AI (например, генерация блоков, подсказки, анализ формул).
- Ответы AI отображаются в AI-панели, с возможностью вставить результат в редактор или скопировать.
- Для работы используется API-ключ Mistral AI (указать в настройках).
- Все действия с AI логируются для аудита.
