# Workflow проекта

## Основные процессы

### 1. Создание калькулятора
- Открыть `/editor`. Собрать схему из блоков (input, formula, data_table и др.)
- Настроить отчёт в панели справа (ReportPanel)
- «Сохранить (Ctrl+S)» — черновик в localStorage

### 2. Отправка на ревью
- «Отправить на ревью» — ввести название и slug (адрес)
- Калькулятор получает статус `review`
- После отправки `current-id` сбрасывается — следующая отправка создаст новый калькулятор

### 3. Ревью админом
- Открыть `/admin/review` (в режиме админа: `/?admin=1`)
- Фильтр «На ревью» → выбрать калькулятор → превью
- Кнопки: «Опубликовать», «Отклонить», «Вернуть на ревью»

### 4. Публикация
- После одобрения статус → `published`
- Калькулятор доступен по `/calculators/{slug}` или `/calculators/{id}`
- Появляется в списке на `/calculators`

### 5. Редактирование
- В ревью-панели: загрузить калькулятор → редактировать → повторно отправить
- Или в редакторе: «Загрузить» → выбрать → внести правки → «Отправить на ревью»

## Пошаговый сценарий

> Все данные — в localStorage одного браузера. Для разных устройств нужен бэкенд.

1. **Главная** `/` → кнопки навигации
2. **Редактор** `/editor` → собрать блоки + отчёт
3. **Черновик** «Сохранить» → статус `draft`, появляется в «Загрузить»
4. **На ревью** «Отправить на ревью» → название + slug → статус `review`
5. **Панель ревью** `/admin/review` → фильтр «На ревью» → «Опубликовать»
6. **Публичная ссылка** `/calculators/{slug}` → калькулятор с полями и отчётом
7. **Список** `/calculators` → все опубликованные калькуляторы

## Создание нового калькулятора

- Нажать «Пустой» — сбрасывает схему и `current-id`
- Или «Сбросить на демо» — загружает демо-данные
- Следующее сохранение/отправка создаст калькулятор с **новым ID**

## Импорт из JSON / ИИ

1. Попросить ИИ сгенерировать JSON по инструкции `AI_CALCULATOR_PROMPT.md`
2. Скопировать результат
3. В редакторе: «Вставить JSON» → вставить → блоки и/или отчёт обновятся
4. Проверить, отредактировать → «Отправить на ревью»

## Технический workflow

- Деплой: push в main → автодеплой на Cloudflare Pages
- Тесты: `npm test` (Vitest, 53 теста)
- Сборка: `npm run build` → dist/
