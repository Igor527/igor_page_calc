## Кто такой админ и как войти

- **Без Firebase** (по умолчанию): админ — любой, кто откроет `/welcome_me`. Флаг сохраняется в localStorage. Выход — «Выйти из режима админа» или `/?admin=0`.
- **С Firebase**: в `.env` заданы `VITE_FIREBASE_API_KEY`, `VITE_FIREBASE_AUTH_DOMAIN`, `VITE_FIREBASE_PROJECT_ID`, `VITE_FIREBASE_APP_ID` и `VITE_ADMIN_EMAIL` (ваш email или несколько через запятую). Тогда по «Войти в режим админа» открывается форма входа; админом считается только пользователь с указанным email.

## Проверка функций после деплоя (вручную)

### Главная и админка
- [ ] Открыть `/` — видна главная, кнопки (Редактор, Калькуляторы, Блог и т.д.)
- [ ] Открыть `/welcome_me` — форма Firebase или сразу переход в режим админа (появится «Выйти из режима админа», блок «Синхронизация с GitHub»)
- [ ] В режиме админа на главной видны ссылки: Словарь, Заметки (админ)
- [ ] «Выйти из режима админа» или `/?admin=0` — выход, ссылки Словарь/Заметки исчезают

### Редактор и калькуляторы
- [ ] `/editor` — редактор открывается, можно добавить блоки, сохранить черновик, нажать «Опубликовать» (ввод названия и slug)
- [ ] После публикации калькулятор появляется в списке на `/calculators` (если есть данные из репо или localStorage)
- [ ] `/calculators` — список калькуляторов; клик по калькулятору — открывается расчёт

### Блог
- [ ] `/blog` — список постов (из репо или пусто)
- [ ] В режиме админа: кнопки «+ Новый пост», «Экспорт для GitHub»; можно создать/редактировать пост, сохранить
- [ ] Открытие поста по ссылке `/blog/:slug` — пост отображается

### Заметки (только админ)
- [ ] Без админа: `/admin/notes` — сообщение «Доступ только для админа»
- [ ] В режиме админа: `/admin/notes` — заметки, можно создать, редактировать, экспорт для GitHub

### Словарь (только админ)
- [ ] Без админа: `/dictionary` — «Доступ только для админа»
- [ ] В режиме админа: словарь открывается, перевод (если задан Mistral — «Проверить доступ к Mistral»)

### Порядок окон (главная)
- [ ] В режиме админа на главной — кнопка ✏️ (редактировать страницу); изменить порядок секций, «Сохранить» — порядок сохраняется

### Синхронизация с GitHub (опционально)
- [ ] В режиме админа на главной открыть «Синхронизация с GitHub», ввести owner, repo, ветку, PAT, «Сохранить», «Проверить подключение» — успех
- [ ] После изменений в заметках/блоге/словаре/калькуляторах/порядке окон изменения автоматически пушатся в репо (если синхронизация настроена)
