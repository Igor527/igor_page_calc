# Ограничения проекта Igor Page Calc

## Ограничения производительности

### Таблицы данных (data_table)

**Максимальный размер таблицы: 500 строк**

- Таблицы с более чем 500 строками автоматически обрезаются до 500 строк при обработке
- При импорте/сохранении таблиц с более чем 500 строками будет показано предупреждение
- Валидация блокирует сохранение таблиц с более чем 500 строками

**Почему это ограничение:**
- Обеспечивает производительность приложения
- Предотвращает проблемы с памятью в браузере
- Ускоряет пересчёт формул

**Что делать, если нужна большая таблица:**
- Обратитесь к разработчику для увеличения лимита
- Рассмотрите возможность разбиения данных на несколько таблиц
- Используйте внешние источники данных (если будет реализована интеграция)

**Где это проверяется:**
- `src/lib/validation.ts` - валидация при сохранении
- `src/lib/engine.ts` - автоматическое обрезание при обработке
- `src/components/editor/PropertyEditor.tsx` - предупреждение в редакторе

## Ограничения формул

### Максимальное количество итераций: 10

- Формулы пересчитываются итеративно до 10 раз
- Если после 10 итераций формулы всё ещё изменяются, пересчёт останавливается
- Это предотвращает бесконечные циклы

**Что делать при превышении:**
- Проверьте циклические зависимости
- Упростите формулы
- Разбейте сложные формулы на несколько блоков

## Ограничения localStorage

### Размер данных

- localStorage имеет ограничение ~5-10 МБ в зависимости от браузера
- Большие калькуляторы могут не сохраниться полностью
- Рекомендуется экспортировать схемы в JSON файлы для резервного копирования

## Ограничения браузера

### Производительность

- При большом количестве блоков (>100) может замедлиться рендеринг
- Используется debounce для ввода значений (300ms задержка)
- Мемоизация пересчётов для оптимизации

## Рекомендации

1. **Для больших таблиц:** Разбейте на несколько таблиц или используйте внешние источники
2. **Для сложных формул:** Разбейте на несколько блоков формул
3. **Для больших калькуляторов:** Регулярно экспортируйте схемы в JSON
4. **Для производительности:** Используйте группы для организации блоков
