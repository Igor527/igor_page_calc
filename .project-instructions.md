# Инструкции проекта Igor Page Calc

## Цель

Веб-конструктор инженерных калькуляторов. Пользователь собирает калькулятор из блоков, отправляет на ревью, админ публикует — калькулятор доступен по публичной ссылке.

## Архитектура

### Блоки — основа всего

Калькулятор — это массив блоков (`Block[]`). Каждый блок имеет `id`, `type` и специфичные поля.

**16 типов блоков** (см. `src/types/blocks.ts`):
`input`, `constant`, `formula`, `data_table`, `table_lookup`, `table_range`, `select_from_table`, `select_from_object`, `condition`, `group`, `output`, `image`, `button`, `table_viewer`, `text`, `chart`

### Поток данных

```
Пользователь вводит → store.updateValue(id, value)
  → recalculateValues(blocks, inputs)   // src/lib/engine.ts
  → Движок вычисляет ALL формулы (по dependencies)
  → store.values обновляется → React перерисовывает
```

Все вычисления — через `recalculateValues()`. Никогда не обходить движок.

### Стейт (Zustand)

`src/lib/store.ts` — единый источник:
- `blocks[]` — схема (блоки)
- `values{}` — вычисленные значения (ключ = id блока)
- `setBlocks()`, `updateValue()`, `setValues()`

Блоки иммутабельны — обновляем через `.map()`:
```typescript
setBlocks(blocks.map(b => b.id === id ? {...b, label: newLabel} : b));
```

### Хранение (localStorage)

`src/lib/calculatorStorage.ts`:
- Каждый калькулятор — ключ `calc-{id}` в localStorage
- Список строится динамически по сканированию ключей `calc-*`
- Статусы: `draft` → `review` → `published` / `rejected`
- Slug (адрес): пользовательский или автоматический (`/1`, `/2`, ...)

### Движок вычислений

`src/lib/engine.ts` — порядок обработки:
1. `INPUT`, `CONSTANT` — начальные значения
2. `DATA_TABLE` — загрузка таблиц
3. `SELECT_FROM_TABLE`, `SELECT_FROM_OBJECT` — выбор из данных
4. `TABLE_LOOKUP`, `TABLE_RANGE` — поиск по таблицам
5. `CONDITION` — условная логика
6. `FORMULA` — итеративный расчёт (до 10 циклов, с детекцией циклов)
7. `OUTPUT` — форматированный вывод

Формулы выполняются через `math.js evaluate()` с безопасным scope. Дополнительные функции: `round`, `roundup`, `rounddown`, `mgnEnlarged`.

### Отчёт (reportHtml)

HTML-шаблон в поле `reportHtml` бандла. Подстановки токенов:
- `@id` — значение блока
- `@id.stepsCalculations` / `@id:exprOnly` — выражение с подставленными числами (без `round`)
- `@id:expr` — выражение = значение

Логика — в `src/lib/reportHtml.ts`.

## Безопасность

`src/lib/security.ts`:
- `sanitizeHtml()`, `escapeHtml()` — XSS защита
- `isValidUrl()`, `sanitizeUrl()` — валидация URL
- `isValidFormula()` — запрет eval, import, __proto__ в формулах
- `containsDangerousCode()` — детектор опасного кода

`src/lib/validation.ts`:
- `validateBlocks()` — полная валидация схемы
- `validateImportedBlocks()` — валидация импортированного JSON
- Проверка зависимостей, циклов, обязательных полей

## Правила для Copilot / AI

- Все комментарии и коммуникация — на русском
- Иммутабельные обновления блоков (`.map()`, spread)
- Все вычисления — через `engine.ts`, не дублировать логику
- Новые типы блоков: добавить в `types/blocks.ts` → `engine.ts` → `PropertyEditor.tsx`
- При генерации JSON калькуляторов — см. `AI_CALCULATOR_PROMPT.md`
