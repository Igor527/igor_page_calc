# Инструкции для разработки и Copilot

## Цели проекта
- Веб-конструктор калькуляторов с возможностью публикации и ревью.
- Гостевой доступ без регистрации (через cookie/localStorage).
- Админ-панель для ручного ревью и управления калькуляторами/блоками.
- Публикация калькуляторов после ревью.
- Хостинг на Cloudflare Pages.

## Основные сущности
- Калькулятор (Calculator): схема, набор блоков, автор, статус (draft, review, published, rejected).
- Блок (Block): тип, параметры, связи, валидаторы.
- Пользователь (User): гость (cookie), автор, админ.
- Ревью (Review): статус, комментарии, история изменений.

## Бизнес-логика
- Гости могут создавать и сохранять калькуляторы локально (cookie/localStorage).
- Для публикации — отправка на ревью (ручное одобрение админом).
- После публикации калькулятор доступен по публичной ссылке.
- Админ может отклонять, комментировать, править калькуляторы.
- Каждый блок проходит валидацию (автоматическую и ручную).

## Паттерны и рекомендации
- Все типы и схемы — в src/types.
- Логика редактора — в src/components/editor.
- Расчёты — в src/lib/engine.ts (math.js).
- Состояние — в src/lib/store.ts (Zustand).
- Примеры схем — в src/data.
- API-роуты (если появятся) — в src/api.
- Документировать сложные места через JSDoc и комментарии.
- Обновлять .project-structure.md при изменениях.

## Ревью и публикация
- Калькулятор отправляется на ревью через UI.
- Админ видит список на ревью, может принять/отклонить/отправить комментарии.
- После одобрения — публикация и доступ по ссылке.

## Гостевой доступ
- Состояние калькулятора хранится в cookie/localStorage.
- Без регистрации — пока cookie не удалены, калькулятор доступен для редактирования.

- Описывать новые паттерны и бизнес-логику в этом файле.

## Оформление страниц (UI/UX)

- Для всех страниц используется PicoCSS (cdn.jsdelivr.net/npm/@picocss/pico@2/css/pico.min.css).
- Базовая структура: фиксированные header (сверху) и footer (снизу), основной контейнер по центру (max-width: 750px).
- Кнопки навигации и действий — крупные, контрастные, размещаются в сетке или flex-контейнере, всегда на одних и тех же местах.
- Цветовая схема: светлая и тёмная тема, по умолчанию — тёмная тема.
- Переключатель темы (DARK/LIGHT) — в header справа.
- Контакты (Telegram, GitHub) — в footer по центру.
- Все новые страницы должны повторять эту структуру и стилизацию для единообразия и масштабируемости UI.
- Для масштабирования: выносить header/footer/контейнер в отдельные компоненты или шаблоны, чтобы переиспользовать их во всех новых страницах.
