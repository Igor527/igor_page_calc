# Защита админки через Cloudflare Access

Админку (редактор и панель ревью) можно закрыть через **Cloudflare Access** (Zero Trust): без паролей в коде, всё настраивается в панели Cloudflare.

## Что защищаем

- **`/editor`** — редактор калькуляторов  
- **`/admin/review`** — панель ревью (одобрение/отклонение, комментарии)

Публичные страницы **`/calculators`** и **`/calculators/:id`** (или **`:slug`**) остаются доступны всем без входа.

---

## Шаг 1: Включить Cloudflare Access

1. Зайдите в [Cloudflare Dashboard](https://dash.cloudflare.com) → ваш домен.
2. В левом меню: **Zero Trust** (или **Access** в старом интерфейсе).
3. Если Zero Trust ещё не включён — нажмите **Get started** и привяжите аккаунт (бесплатный план подойдёт).

---

## Шаг 2: Создать приложение (Application)

1. В Zero Trust: **Access** → **Applications** → **Add an application**.
2. Выберите **Self-hosted**.
3. Настройки:
   - **Application name:** например `Igor Page Calc — редактор и админка`
   - **Session Duration:** по желанию (например 24 hours)
   - **Application domain:**
     - **Subdomain:** ваш домен, на котором крутится сайт (например `igor-page.pages.dev` или `calc.yourdomain.com`)
     - **Path:** укажите пути, которые нужно защитить:
       - Вариант А: только админка — путь **`/admin`** (будут защищены `/admin` и `/admin/review`).
       - Вариант Б: редактор и админка — путь **`/`** (будет защищён весь сайт, включая редактор; тогда для публичных калькуляторов см. ниже).
4. Нажмите **Next**.

---

## Шаг 3: Политика входа (Policy)

1. **Policy name:** например `Разрешить только мне` или `Команда редакторов`.
2. **Action:** **Allow**.
3. **Configure rules:**
   - **Include:**
     - **Emails** — укажите свой email (или несколько через запятую), с которых разрешён вход,  
       **или**
     - **Emails ending in** — например `@yourdomain.com`,  
       **или**
     - **One-time PIN** — любой, у кого есть ссылка, сможет ввести код из письма (удобно для гостевых ревьюеров).
4. **Next** → **Save**.

В итоге доступ к выбранным путям получат только те, кто прошёл проверку (email или OTP).

---

## Важно: если защитили весь сайт (Path = `/`)

Если в шаге 2 вы указали путь **`/`**, то и публичные страницы `/calc/:id` будут за Access. Тогда нужно добавить **второе приложение**:

1. **Add an application** → Self-hosted.
2. **Application domain:** тот же Subdomain, **Path:** **`/calc`**.
3. В политике выберите **Bypass** (или **Allow** с правилом «Everyone») — чтобы `/calc/*` были доступны без входа.

Или оставить одно приложение с путём только **`/admin`** — тогда редактор (`/`) остаётся открытым, защищена только панель ревью. Решите по своей модели доступа.

---

## Результат

- При заходе на защищённый путь пользователь увидит страницу входа Cloudflare (email или OTP).
- После успешной проверки попадёт в редактор или в панель ревью.
- Публичные калькуляторы по `/calc/:id` (если не включены в защищённый путь) работают без входа.

Изменений в коде приложения не требуется — защита полностью на стороне Cloudflare.
